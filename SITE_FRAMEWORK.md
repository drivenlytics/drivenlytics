# Drivenlytics Build Framework
## The 6-Level Methodology

---

## The Six Build Framework Levels

Drivenlytics projects follow six core areas for building systems and projects:

1. **Level 1: Setup & Project Organization**
2. **Level 2: Performance**
3. **Level 3: Speed**
4. **Level 4: Better Design**
5. **Level 5: Integrations & Deployment**
6. **Level 6: Cost Savings**

---

## Level 1: Setup & Project Organization

### Core Rule
**One project folder per one URL.** If it has a separate URL, create a new project.

### Step 1: Create Anti-Gravity Skills Folder
- Create a `anti-gravity skills` folder
- Import this folder into all future projects
- Keep all reusable skills here (design skills, audit skills, etc.)
- Accessible at any stage of any build

### Step 2: Extensions & Customizations
- Click on extensions (like Chrome extensions for the tool)
- Install extensions that change presentation and visuality
- Monitor this section regularly

### Step 3: Rules & Workflows
**Rules:**
- Global document that governs everywhere
- Added to context window for every message
- Increases token spend
- Examples: coding style preferences, specific design instructions
- Write anything you want the system to know about your practice

**Workflows:**
- Different from skills
- Instructions for specific actions (e.g., debugging workflow)
- Create and name as needed
- Access via forward slash `/` command
- Not constantly in context—summoned when needed
- Universal access without context overhead

### Step 4: File Organization
- Right-click and create new folders
- Use `@` sign to call out specific files in chat
- List multiple files as needed

### Step 5: Version Control with GitHub
- Regularly publish updates to GitHub
- Version control lets you roll back any breaking change
- If something breaks, you can restore it as if it never happened
- Commit and confirm changes with the system

### The Blast Master System Framework
Use this framework for everything you build (website, app, or other project).

**Initialization Protocol (Protocol Zero):**
- Create `task_plan.md` — phases, goals, checklists
- Create `findings.md` — research, discoveries, constraints
- Create `progress.md` — what's been done, errors, tests, results
- Create `gemini.md` or `project-constitution.md` — project constitution

**Blueprint Phase (Phase 1) — Vision and Logic:**
- What do you want to include?
- What is your singular desired outcome in plain English?
- What integrations do you need?
- Sources of truth (where does data live)?
- Delivery payload (what does "done" look like)?
- Behavioral rules (how should it behave)?
- Apply the **"data first" rule**

**Links Phase (Phase 2) — Connectivity:**
- What external services does it need to connect to?
- Test all linkages and verification handshakes
- Confirm all connections exist before proceeding

**Architectural Layer (Phase 3) — Build:**
- Build the minimal viable product
- Three-step process for core functionality

**Stylization Layer (Phase 4) — Design:**
- Refine the UI
- Make sure you're happy with the design

**Trigger Phase (Phase 5) — Deployment:**
- How does it run?
- Is it running in the cloud?
- Detail all deployment specifics

### Operating Principles
- **Data First Rule** — Define schemas before building
- **Self-Healing Repair Loops** — Model repairs itself and improves over time
- **Deliverables vs. Intermediates** — Know the difference
- **File Structure Reference** — Maintain clean organization

## Level 2: Performance

### Context Rot Prevention
- Performance degrades after halfway through a long conversation
- Once a task is completed, start a new window
- Keep multiple conversation windows for different projects
- Return to any previous conversation as needed

### Alternative Access Methods
1. **Double-click into pane** to access Claude plugin
2. **Terminal access** — Type `claude` after installation
   - Create infinite terminal instances
   - Position them anywhere

### Agent Manager
- Access via Agent Manager button
- Jump between different project conversations instantly
- Create new conversations in any section with `+` button
- Select and switch between projects at will
- No performance penalty for conversation switching

## Level 3: Speed

### Inbox Management
- Inbox shows all approvals needed
- Review and approve multiple items quickly
- Create new conversations in any folder
- Create unlimited folders for organization

### Playground Feature
- Click "use playground" from any conversation
- Blank canvas for new research or building
- Fresh context window
- Bring files over to other conversations
- Much more productive output

### Sub-Agent Feature
- Open terminal and request sub-agent feature
- Example: "Spin up five sub-agents, each researching a capital city and writing 500-word essays"
- All sub-agents run simultaneously in parallel
- Separate context windows for each
- System orchestrates and manages them

### One Task Per Message Rule
- One message = one task
- Do not combine multiple requests in one message
- Example: Create one section of website per message (not multiple changes at once)
- Results in faster performance and higher quality output

### Breaking Loops
If stuck in a loop where the system repeats itself:
- **Prompt:** "We've been going in a loop. Stop and explain succinctly but in detail exactly how you're going to solve this. What is going to be different this time than last time?"
- Alternative: Start a brand new window

### Visual Intelligence / Logic Flow System
- Use visual tools to verify system logic before building
- Prevents building systems without understanding the flow
- Example: Build flow diagrams to verify scraper logic before deployment
- Free tools available (e.g., pizza shop logic system)

**Pizza Shop Logic Example Process:**
1. Describe the intelligent system you want
2. System creates requirement doc and blueprint
3. Access the logic system universally (like MCP)
4. Run the logic through sequence
5. Yellow elements represent decision points
6. Modify variables and test outcomes
7. Share and integrate into your MCP config

### MCP Integration into Config
1. Click share on logic system
2. Copy standard endpoint
3. In editor, paste endpoint with API key
4. Test endpoint: "Test these endpoints and tell me what variables we can send"
5. Send variables to system and get answers back
6. Add to MCP config when satisfied

### Skill Creation
- If you do something you'll repeat 2-3+ times per month, turn it into a skill
- Use anti-gravity skills creator
- Copy and create new skill
- Example: GitHub publish + Vercel deploy combined skill
- Paste into anti-gravity skills folder

## Level 4: Better Design

### AI Studio for Design
- Build beautiful designs in AI Studio (front-end of the system)
- AI Studio has access to more design tools out of the box
- Download and import into code environment
- Get to 80% of desired design, then refine with techniques

### Design Workflow
1. Build something beautiful in AI Studio
2. Look at history of builds
3. Get to 80% of desired state
4. Download the app with appropriate name
5. Open folder in code editor
6. Edit and modify as needed
7. View in localhost: "Hey, open this in localhost for me"

### Design Hack
- AI Studio has more built-in design tools than code environment
- Build in AI Studio for visuals
- Bring into code for scalability and implementation

### Design Inspiration — Start Here
1. Visit `godly.website` — curated gallery of the best web design
2. Browse by category or style to find 2–3 sites that match your vision
3. Screenshot or note the URLs for HTML extraction

### HTML Extraction for Design Inspiration
1. Find a website you like (e.g., from godly.website)
2. Copy URL
3. Go to Google and search "HTML website extractor"
4. Enter URL and click "view source"
5. Download the HTML file
6. Upload to AI Studio
7. Request variation: "Using this HTML, build me a [color] version that looks even more epic and does [something cool]"
8. Extract features and patterns (never sell or give to clients)

### UI Sniping
1. Visit website like `21st.dev` for design inspiration
2. Find CTAs, design aspects you like
3. When you find one you want: Click "share component"
4. Paste into code editor with prompt
5. System fetches link, grabs code, can replicate
6. Alternative: Copy prompt directly (includes all code)

### CodePen for Components
1. Visit `codepen.io`
2. Find cool components (cards, animations, etc.)
3. Copy HTML, CSS, JavaScript
4. Insert into your website
5. Customize as needed

### Converting Sites Between Platforms
1. Build in one platform (e.g., code)
2. Request conversion: "Take this website and turn it into HTML and CSS"
3. Deploy to another platform (e.g., GoHighLevel)

### GoHighLevel Integration
1. Go to `gohighlevel.com`
2. Navigate to Sites → Create new funnel
3. Edit and add elements
4. For code element: Click "add element" → Select "code"
5. Request conversion: "Turn this into HTML and JavaScript"
6. Copy converted code
7. Paste into GoHighLevel code element
8. Website is now live in GoHighLevel

### Website Audit Skill
1. Get the website audit skill (50+ checks)
2. Add to anti-gravity skills folder
3. After building in AI Studio, request audit against skill
4. Checks: SEO optimization, accessibility, contrast, 90+ quality factors
5. Request checklist: "Give me a checklist with green tick for all changes made"

### UI Auditing with Screenshots
1. Command + Shift + 5 (Mac) or equivalent
2. Paste screenshot into chat
3. Discuss directly with screenshot context
4. System can edit and annotate
5. Ensures specific, refined designs

## Level 5: Integrations & Deployment

### Three-Step System for Integration

#### Step 1: Check One-Click MCP Servers
1. Click three dots → "MCP servers"
2. Look for service (e.g., Supabase)
3. If found: Click install button
4. Generate token from service
5. Return and integrate token

#### Step 2: Manual Installation from MCP Market
1. Visit `mcpmarket.com`
2. Search for service
3. Validate reviews on GitHub
4. Go to README and copy example config
5. Return to editor
6. Paste config: "Add this to my MCP config, please"
7. Enter API token

### Recommended MCP Services
- **Apify** — Scrape anything
- **Context 7** — Latest API documentation
- **AI Meeting Note Takers** — Fathom, Fireflies
- **Link to Pizza** — Pizza shop example
- **NotebookLM** — Research tool
- **Notion** — Project management
- **Pinecone** — Long-term memory
- **Stitch** — Multiple integrations
- **Supabase** — Database storage
- **Vercel** — Website deployment
- **Zapier** — Universal connector (8,000+ apps)

### Zapier as Universal Connector
1. Go to `zapia.mmc` or `mcp.zapier.com/servers`
2. Click "add tool"
3. Select from 8,000+ apps
4. Example: Gmail gives access to all email functions
5. Actions: Get attachments, find, add, archive, delete, create, send
6. Click "connect" → Opens authorization
7. Select Google account
8. Approve access level
9. Add all desired tools
10. Return to system: Zapier is now enabled

### Using Zapier in System
- Ask: "What is the subject line of my latest email? Use the Zapier MCP."
- System uses Zapier to check inbox and returns answer
- Can query any connected service

### Vercel Integration
1. Use same MCP process as Zapier
2. Generate token from `vercel.com/account/tokens`
3. Add to MCP config
4. Vercel hosts websites
5. GitHub + Vercel: Automatic deployment to live site

### GitHub and Vercel Publishing
1. One-click publish to GitHub
2. Vercel auto-deploys from GitHub
3. Combine into single skill: "publish [to these services]"
4. Instant upload and deployment

### Modal for Scheduled & Serverless Execution
- Modal is where remote servers run your code
- Use when you need code to run on schedule or when laptop is closed
- Example: Daily website scraper that sends report at 6am
- Runs on Modal's servers regardless of your status

#### Setting Up Modal
1. Create free account at Modal
2. Go to app section → Click quick start
3. Copy code
4. Paste into editor
5. Connects Modal for you automatically
6. Open in new window and authorize

#### Using Modal
1. Schedule tasks (e.g., scraper at 6am daily)
2. Make accessible via HTTP node
3. Integrate with N8N for client workflows
4. Sell the automated service (client calls HTTP endpoint)
5. Client uses your code running on Modal
6. Client can call with natural language parameters

## Level 6: Cost Savings

### Reduce File Size
- Use concise "Queen's English"
- Shorten all prompts
- Summarize frequently

### Starting New Windows for Token Management
- If window grows large, start new one
- Don't copy-paste everything
- Request summary: "Summarize succinctly this conversation: problems and next steps such that a new agent could pick up where you left off. Be concise."
- Take summary, open new conversation, begin fresh

### Model Selection Strategy
- Use powerful models for appropriate tasks only
- Thinking model = planning stage
- "Don't need Shakespeare to cut grass"
- Separate tasks by complexity
- Simple research → simpler model
- Complex thinking → most powerful model
- Save context and costs by matching model to task

### Fast Mode
- Use "fast" mode for most tasks
- Reserve planning for when necessary
- Avoid loops and overkill processing
- Tokens are relevant—use strategically

### MCP Management
1. Click MCP servers → Manage MCP servers
2. Check active MCPs (recommend max 50)
3. Deactivate unused MCPs
4. Use "lazy loader" to reduce context
5. Significantly saves context window

### Alternative Models
- If struggling with context window
- Install "open code"
- Access hundreds of models basically for free
- Select different models for different tasks
- Use subscription for most capable model

### Multiple Account Strategy
- Run out of most powerful model quickly
- Switch between multiple accounts (5+)
- Rotate fresh access to powerful models
- Consider value of your time vs. subscription cost
- Valid strategy if optimizing for speed/power

## Operating Summary

This framework provides systematic progression from setup (Level 1) through full deployment with cost optimization (Level 6). Each level builds on previous foundation. Start at Level 1, progress sequentially through all six levels.

The core principle: structured, intentional building with integration, deployment, and optimization at every stage.
