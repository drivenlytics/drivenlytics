## Level 3: Background Automation
**14:49**
What if you just want your content to run automatically in the background? Let's say you're a content creator who wants to repurpose YouTube videos into blog posts for LinkedIn or X. We can provide the YouTube channel link and say: "I want to create an automation that will run in the background. Whenever Nate uploads a new YouTube video, use potato to get the transcript, summarize it into a blog post, generate an infographic using Nano Banana Pro, and schedule that for posting to LinkedIn."

**15:46**
The automation checks the YouTube channel using an RSS feed. Transcript extraction is handled by Plotato via the MCP. Once it has the transcript, it generates an infographic using a style reference (style.png) we provided. Then it writes the LinkedIn post using a tone of voice file for the creator.

**16:32**
To make it "always on," even if anti-gravity is closed, you deploy it to modal.com. Modal is a great option for deploying automations because of their credit system. The pipeline is scheduled (e.g., at 10:00 a.m. UTC). Every time it triggers, it checks for new videos and runs the full process.
